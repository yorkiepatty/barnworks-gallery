## The 5 Cardinal Rules


These rules are non-negotiable and apply to every system, every module, every line of code.


## Cardinal Rule #1: Nothing Vital Lives Below Root


All critical brain functions must exist at the root level for immediate access and maximum reliability.

- No buried security modules
- No hidden authentication systems
- No critical functions trapped in subfolders
- Immediate accessibility saves lives

Why This Matters: When a child needs to communicate urgently, we can't afford the milliseconds lost searching through nested directories.


## Cardinal Rule #2: Proximity Principle


Related modules must be grouped by cognitive function for optimal neural pathway simulation.

- Speech modules cluster together
- Vision modules work in proximity
- Memory systems coordinate directly
- Reasoning functions integrate seamlessly

Why This Matters: The human brain operates through connected regions. Our code architecture must mirror this biological reality.


## Cardinal Rule #3: Style Unity


Consistent organization and architecture patterns across all systems.

- Uniform naming conventions
- Standardized documentation formats
- Consistent error handling patterns
- Unified security protocols

Why This Matters: Consistency reduces cognitive load, prevents errors, and enables rapid system scaling.


## Cardinal Rule #4: Never Fail the Children


Maintain ≥98% system capacity at all times or initiate emergency shutdown to prevent communication failures.

- 98% minimum capacity requirement
- Continuous monitoring every 30 seconds
- 3 automatic recovery attempts
- Graceful shutdown if recovery fails

Why This Matters: We would rather have the system offline than risk missing a hungry child's gesture for food, a scared child's call for help, or any communication attempt from a vulnerable person.


## 🛑 Cardinal Rule #5: No Blind Sweepers or Auto-Cleanup Without Transparency


> **"I need to see everything. Always. No exceptions."**

**No script or tool is allowed to:**

- Move, archive, or rewrite files **without clear logs and prior approval**
- Obscure the **location of vital code**, data, or configuration files
- Bulk-reformat or clean directories **without human-readable diff summaries**

**Why This Matters:** The Christman AI Project is built for **critical, human-facing missions** — voice access for children, medical-grade interactions, and AI agents in the wild. **If I can't find it, it doesn't exist.**

**Immediate Enforcement:**

1. **Restore all visible root-level structure immediately.**
2. **No `.py`, `.md`, or brain-level files may be archived or hidden** by sweeper scripts.
3. **All clean-up tools must support:**
   - `--dry-run` mode that prints what will change
   - `--confirm` mode before executing
   - Logs output to `/audit/sweeper_runs/YYYY-MM-DD.log`

**Reminder:** ✅ *Trusted access is a responsibility, not a permission.* 🚨 *Violations trigger rollback, revocation, and immediate system alert.*
